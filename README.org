#+TITLE: misemacs
#+AUTHOR: Jonathan Carroll Otsuka

* Overview

misemacs builds multiple Emacs /flavors/ (=<port>-<line>=) — currently
=emacs-master= (mainline GNU Emacs, NS port) and =emacs-mac-master= (the Mac
port) — each a fully relocatable =Emacs.app= for macOS, built end-to-end via
mise. Every C library and build tool comes from conda-forge via mise's conda
backend; mise's experimental =[deps.*]= provider system orchestrates the build
graph (with blake3 content-hash freshness checks).

Three from-source packages handle the emacs-specific glue, plus the
emacs build itself:

- =libs/enchant= — enchant 2.8.3 with =--enable-relocatable= +
  =--with-applespell=, providing libenchant-2.a (statically embedded
  into jinx-mod) and the AppleSpell provider plugin.
- =libs/jinx-mod= — Emacs dynamic module wrapping libenchant for the
  jinx spell-check package.
- =libs/emacs-libvterm= — Emacs dynamic module wrapping libvterm for
  the vterm terminal package.
- =pkgs/<flavor>= — runs Emacs's autogen+configure+make against the
  conda-provided deps and bundles the result into a self-contained
  =Emacs.app= at =build/<flavor>/Emacs.app=.

* Install (recommended)

End users get pre-built =Emacs.app= bundles via mise + aqua.
Requires [[https://mise.jdx.dev][mise]] (>= 2026.x).

#+begin_src bash
# Latest build of a flavor:
mise use -g aqua:djgoku/misemacs/emacs-mac-master@latest
mise use -g aqua:djgoku/misemacs/emacs-master@latest

# Pin an exact build (umbrella package, java-style flavored version):
mise use -g aqua:djgoku/misemacs@emacs-mac-master-2026.05.27

# Browse every flavor + build:
mise ls-remote aqua:djgoku/misemacs

# Expose to Spotlight/Finder/Dock:
ln -snf "$(mise where aqua:djgoku/misemacs/emacs-mac-master)/Emacs.app" ~/Applications/Emacs.app
#+end_src

To uninstall the Spotlight integration: =rm ~/Applications/Emacs.app=.
To uninstall a flavor: =mise rm aqua:djgoku/misemacs/emacs-mac-master=.

* Flavors

| Flavor (= tag prefix) | Port          | Line   | Source                                        |
|-----------------------+---------------+--------+-----------------------------------------------|
| emacs-master          | mainline (NS) | master | emacsmirror/emacs @ master                    |
| emacs-mac-master      | Mac port      | master | jdtsmith/emacs-mac @ emacs-mac-gnu_master_exp |

Releases are tagged =<flavor>-<calver>= (e.g. =emacs-mac-master-2026.05.27=);
each tag is one GitHub release carrying =misemacs-<flavor>-<calver>-macos-arm64.tar.gz=.

* Adding a flavor

A flavor is =<port>-<line>=. To add one:

1. *Source.* Create =pkgs/<flavor>/versions.toml=:
   #+begin_src toml
   [source]
   type = "git"
   repo = "https://github.com/<owner>/<repo>"
   ref  = "<branch-or-tag>"
   #+end_src
   Examples:
   - =emacs-mac-30=:  repo =jdtsmith/emacs-mac=, ref =emacs-mac-30_1_exp=
   - =emacs-31=:      repo =emacsmirror/emacs=,  ref =emacs-31=
   - =emacs-30=:      repo =emacsmirror/emacs=,  ref =emacs-30=
   - =emacs-29=:      repo =emacsmirror/emacs=,  ref =emacs-29=
2. *build.toml.* Copy from the nearest existing flavor (Mac lines copy
   =pkgs/emacs-mac-master=; NS lines copy =pkgs/emacs-master=). Older NS lines
   may need =configure_args= tweaks — build, read the error, adjust.
3. *lockfile.toml.* =mise run bump <flavor> latest=
   (resolves the =ref=) to pin the sha and hydrate.
4. *Build graph.* Add =[deps.pkgs-<flavor>]= to =mise.toml= (copy a sibling,
   repoint =sources=/=outputs=/=run= to =<flavor>=); add =<flavor>= to the case
   and =PKGS= in =scripts/cli/build.sh=, to =PKG_ALIASES= in =scripts/cli/_lib.sh=,
   and to the =PKGS= lists in =doctor.sh= / =status.sh=;
   add it to the =validate= variant loop in =mise.toml=.
5. *CI.* Add =<flavor>= to the =release.yaml= matrix and the
   =nightly-emacs-bump.yaml= bump loop.
6. *Registry.* In the aqua-registry fork add
   =pkgs/djgoku/misemacs/<flavor>/registry.yaml= with
   =version_filter: Version startsWith "<flavor>-"= and
   =version_prefix: "<flavor>-"=; test via =MISE_AQUA_REGISTRY_URL=; PR upstream.
7. *README.* Add a row to the Flavors table above.

* Build from source (contributors)

If you want to build =Emacs.app= yourself instead of installing a
release tarball:

#+begin_src bash
mise run bootstrap                                  # mise install + hydrate
mise run build emacs-master                         # builds with self-healing precheck
mise run validate                      # 8/8 doctor + bundle self-contained gate

cp -R build/emacs-master/Emacs.app ~/Applications/Emacs.app
#+end_src

Substitute =emacs-mac-master= (or any other flavor) for =emacs-master= as appropriate.

* Verify

Run both checks with a single command:

#+begin_src bash
mise run validate
#+end_src

Or run them individually:

#+begin_src bash
# Static analysis: every Mach-O reference inside the bundle must be
# either @-relative or under /System / /usr/lib. Catches absolute
# conda-mise paths leaking into the bundle.
bash scripts/verify-bundle-self-contained.sh build/emacs-master/Emacs.app

# Runtime: load jinx-mod + vterm-module, confirm enchant returns
# at least one language. (Inside Emacs: M-x moto-emacs-doctor)
build/emacs-master/Emacs.app/Contents/MacOS/Emacs -batch \
  -l moto-emacs-doctor -f moto-emacs-doctor-batch
#+end_src

Both should exit 0; the doctor reports 8/8 PASS.
Substitute =emacs-mac-master= (or any other flavor) as appropriate.

* Architecture

#+begin_src
mise.toml
├── [tools]               — perl, python, conda:* (link-time deps + build tools)
├── [tasks.*]             — user-facing verbs: bootstrap, build, bump,
│                            doctor, status, validate, hydrate, test-cli, …
└── [deps.*]              — experimental mise providers; the build graph
    ├── conda-glib …      — scripts/build/conda-prefix.sh <X>
    │                        writes build/conda-<X>/{lib,include,share,bin,.path}
    ├── libs-enchant      — scripts/build/autotools.sh libs/enchant
    ├── libs-jinx-mod     — scripts/build/cc-module.sh libs/jinx-mod
    ├── libs-emacs-libvterm
    │                     — scripts/build/cc-module.sh libs/emacs-libvterm
    └── pkgs-<flavor>     — scripts/build/emacs-app.sh pkgs/<flavor>

scripts/build/
├── _lib.sh               — shared helpers: resolve_dep_prefixes,
│                            tool_dep_path_export, dep_injection_export,
│                            rpath_install_name
├── conda-prefix.sh       — symlinks a conda env's subdirs into
│                            build/conda-<X>/, writes a .path manifest
├── autotools.sh          — ./configure → make → make install (DESTDIR pattern)
│                            from <pkg-dir>/build.toml
├── cc-module.sh          — compiles a single .so from C sources, with
│                            optional pkg-config + static-archive linking
└── emacs-app.sh          — runs Emacs's autogen+configure+make and bundles
                             a relocatable .app into build/<flavor>/Emacs.app

libs/<X>/
├── build.toml            — per-target build metadata (linkage, configure_args,
│                            module_srcs, lib_deps, tool_deps, companion scripts)
├── lockfile.toml         — sha pin for the from-git source
├── versions.toml         — repo URL, source.type=git, optional submodules=true
├── pre-autogen.sh        — optional companion (e.g., libs/enchant)
└── bootstrap.sh          — optional companion (e.g., libs/enchant)

pkgs/<flavor>/
├── build.toml            — emacs-specific knobs (configure flags, bundle_*,
│                            *_dep names, lib_deps, tool_deps)
└── (same lockfile.toml, versions.toml shape as libs/)

scripts/
├── hydrate.sh                          — populate libs/<X>/src + pkgs/<X>/src worktrees
└── verify-bundle-self-contained.sh     — relocatability gate
#+end_src

   ([tools] conda:* …)              ([deps.*] graph)

        ▼                                 │
   mise install                           │
   conda libs/tools at $HOME/.local/      │
        │                                 │
        ▼                                 │
   scripts/hydrate.sh                     │
   git worktrees + src-sha.txt            │
        │                                 │
        └──────────────┬──────────────────┘
                       ▼
                 mise run build <flavor>
                       │
                       ├── [tasks.build] → scripts/cli/build.sh
                       │       │
                       │       └── self-healing precheck (lockfile coherence,
                       │             worktree HEAD, conda env presence)
                       │
                       └── mise deps install pkgs-<flavor>

   ┌────────────────────────────────────────────────────────┐
   │ scripts/verify-bundle-self-contained.sh                │
   │   LC_LOAD_DYLIB + LC_RPATH must be @-relative or       │
   │   under /System / /usr/lib                             │
   │                                                        │
   │ moto-emacs-doctor-batch                                │

* What's in Emacs.app

Bundle layout, relative to =Emacs.app=:

#+begin_src
Contents/
├── MacOS/
│   ├── Emacs                              ← the binary
│   ├── bin/{etags,emacsclient,ebrowse}    ← ship-along utilities
│   └── libexec/{Emacs.pdmp,hexl,...}      ← internal helpers
├── Frameworks/
│   ├── lib/                                ← every linked dylib
│   │   ├── libgnutls.30.dylib              ← from conda
│   │   ├── libtree-sitter.0.26.dylib       ← from conda
│   │   ├── libpng16.16.dylib               ← from conda
│   │   ├── libgmp.10.dylib                 ← from conda
│   │   └── …
│   │   └── enchant-2/
│   │       └── enchant_applespell.so       ← from libs/enchant
│   └── share/enchant-2/AppleSpell.config   ← AppleSpell runtime config
└── Resources/
    ├── build-manifest.org              ← build provenance: emacs SHA, conda versions,
    │                                       configure flags, bundled dylib list
    └── site-lisp/                          ← on Emacs's load-path
        ├── jinx-mod.dylib                  ← (require 'jinx) → load chain
        ├── jinx.el
        ├── vterm-module.dylib              ← (require 'vterm) → load chain
        ├── vterm.el
        ├── moto-emacs-doctor.el            ← M-x moto-emacs-doctor
        └── site-start.el                   ← autoloads doctor
#+end_src

Note: =libenchant-2= is NOT in =Frameworks/lib/= — its code is statically
linked into =jinx-mod.dylib=.

The =Frameworks/lib/= layout matches what gnulib's =--enable-relocatable=
expects (=lib/<basename>=), but the actual relocation work is done by
=relocate-init.c= inside =jinx-mod=, because gnulib's auto-detect is
Linux-only on the macOS path. See =libs/jinx-mod/relocate-init.c= for
the dladdr-based runtime prefix shim.

* Known gotchas

The previous list of "remember to also run X" items has been
systemically closed:

- =mise run build <flavor>= runs a self-healing precheck: on drift between
  =lockfile.toml=, =pkgs/<flavor>/src/=, and =src-sha.txt=, it auto-runs
  =scripts/hydrate.sh <flavor>= and re-verifies before compiling.
- Heavy =[deps.libs-*]= and =[deps.pkgs-emacs]= providers use =auto =
  false=, so cheap mise verbs (=hydrate=, =doctor=, =status=, =bump=)
  never trigger an Emacs compile during their prelude. Only
  =mise run build= and =mise run validate= compile.
- =[deps.conda-*]= providers stay =auto = true= (symlink-only,
  millisecond cost).
- =src-sha.txt= is the byte-deterministic sha sentinel in =[deps.*]=
  sources; =lockfile.toml= is intentionally not listed there, so a
  bump → bump-back to the same sha is a cache hit instead of a phantom
  rebuild.
- The builder scripts themselves (=emacs-app.sh=, =autotools.sh=,
  =cc-module.sh=) call =assert_pkg_coherence= in
  =scripts/build/_lib.sh=, which performs the same self-healing step.

If something looks off, run =mise run doctor= for a cross-layer invariant
report with per-failure remedies. (Doctor is read-only; it reports drift
without auto-fixing, so the user can distinguish "actually drifted now"
from "drifted but auto-fixed during the last build.")

** Editing =lockfile.toml= by hand

The canonical bump path is =mise run bump <pkg> <sha>= — it rewrites
the lockfile and re-hydrates the worktree atomically.

If you instead edit =lockfile.toml= directly (e.g., =vim
pkgs/emacs-master/lockfile.toml=), the next =mise run build <flavor>=
self-heals: the precheck detects the drift between =lockfile.toml= and
=pkgs/<flavor>/src/=, auto-runs =scripts/hydrate.sh= for the affected pkg,
re-verifies coherence, then proceeds with the build. Single verb, no manual
hydrate step required.

#+begin_src bash
# Edit pkgs/emacs-master/lockfile.toml by hand, then:
mise run build emacs-master           # auto-hydrates + builds the new sha
#+end_src

The auto-hydrate emits a =mise run build: <pkg>: drift (...) —
auto-hydrating …= line so the action is visible in the build log.

* Fallback: bash-only baseline

A pure-bash variant that produces the same =Emacs.app= lives at the
=brainstorm/mise-conda-emacs= branch (commit =736e01b=). Five scripts
under =scripts/conda-emacs/= chain =mise install= → =hydrate= →
autogen/configure/make → bundle. Useful as a reference if anything
breaks, or as a starting point for a different orchestrator. Validated
via the same =moto-emacs-doctor= 7/7 gate.

* Releases

Tags are =<flavor>-<calver>= (e.g. =emacs-mac-master-2026.05.27=) and are
immutable. Each tag is one GitHub release carrying:

- =misemacs-<flavor>-<calver>-macos-arm64.tar.gz= — the bundle
- =SHASUMS256.txt= — verify with =shasum -a 256 -c=
- =build-manifest.org= — full provenance (emacs SHA, conda library
  versions, configure flags)

The move to flavored tags was a clean cutover; the prior bare-CalVer
release scheme is superseded (old bare tags may be deleted).

A Sigstore artifact-provenance attestation is published with each
tarball; verify with =gh attestation verify ... --owner djgoku=.

If a release is broken, do *not* retract — supersede. Publish a new
flavored CalVer release; users move forward by version pin.
